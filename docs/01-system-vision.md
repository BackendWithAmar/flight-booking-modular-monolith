# 01 – System Vision

## Project Name

Flight Booking Platform Backend

## Objective

The goal of this project is to design and implement a **production-style backend system** that simulates how large-scale travel booking platforms operate.

Examples of similar systems include global travel platforms that support millions of users searching and booking flights daily.

The platform will allow users to:

* Search flights
* Create bookings
* Process payments
* Earn rewards
* Receive notifications

The system will be developed using **Java and the Spring ecosystem**, with a focus on learning **real backend engineering practices**.

---

Backend Frameworks

* PostgreSQL

---

## System Capabilities

The final system will demonstrate the following backend engineering concepts:

* REST API architecture
* Authentication and authorization
* Database modelling
* Event-driven communication
* Distributed system design
* Observability and monitoring
* Scalable infrastructure

---

## Architecture Evolution

The system will evolve through multiple architectural stages:

Monolith
↓
Modular Monolith
↓
Microservices
↓
Event-Driven Architecture

This mirrors how many real-world backend systems evolve in production environments.
